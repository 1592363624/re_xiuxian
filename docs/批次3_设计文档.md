## 一、设计哲学与总体调性

### 1.1 核心调性：慢养成、强策略、长线闭环

- **慢养成**：飞升是一条 30~60 天级别的长线，第二元神 / 小世界 / 法则均按"日积月累"推进，不允许氪金一次性跳级
- **强策略**：飞升前置条件（大衍诀五层·衍神 + 空间节点 + 法则材料）必须由玩家在不同玩法中分别完成，鼓励多系统联动
- **长线闭环**：第二元神 → 小世界 → 神庙 → 香火 → 神识淬炼 → 法则 → 飞升，构成首尾相连的资源环
- **死亡有代价**：夺舍重生作为"高阶修士的二次机会"，必须保留境界跌落 + 属性损失 + 冷却三重代价，避免死亡无意义化

### 1.2 数值膨胀控制原则

| 维度 | 上限设计 | 设计意图 |
|------|---------|---------|
| 飞升灵界境界要求 | 化神巅峰（rank≥20） | 防止低境界玩家借助材料强行飞升 |
| 大衍诀有效神识倍率 | x3.00（五层·衍神） | 与世界 Boss / 高阶副本挂钩 |
| 第二元神属性继承比 | 主元神 60% | 防止双元神叠加导致属性翻倍 |
| 小世界人口上限 | 100,000 / 神庙等级 10 | 香火产出有硬上限 |
| 香火→神识兑换比 | 1 神识 = 100 香火 | 防止香火通胀直接灌满神识 |
| 双修修为加成 | 单次 +5%，日上限 3 次 | 单身玩家不被碾压 |
| 侍妾远航收益上限 | 单次最多 3 件稀有材料 | 防止远航刷材料 |
| 掩月抢亲队伍人数 | 3~5 人（队长 1 + 队员 2~4） | 剧情副本，人多为乱 |
| 端午镇蛟队伍人数 | 固定 10 人 | 强制协作，缺一不可 |
| 夺舍境界跌落 | 至少跌 1 大境界 | 死亡必须有痛感 |

## 三、飞升 + 夺舍重生系统设计

### 3.1 玩法概述

参考玩法文档第18节"神识、太一门与空间节点"和第33节"大衍诀、神识与飞升"，飞升是元婴后期 → 化神 → 灵界的终极跨越。流程：

1. **前置条件**：化神巅峰（rank=20）+ 大衍诀五层·衍神 + 收集空间法则碎片 + 凝聚逆灵通道坐标
2. **空间节点准备**：`.搜寻节点` → 发现不稳定节点 → `.定星 <节点名>` 稳固节点 → 成功后获得逆灵通道坐标 / 虚灵丹丹方 / 万年灵乳 / 空间法则碎片
3. **正式飞升**：`.飞升灵界` → 系统校验全部前置 → 触发神识反噬判定 → 成功则飞升灵界、失败触发虚弱+残魂
4. **失败处理**：残魂<30触发不稳定 → 玩家可 `.天机回溯` 尝试回溯一次 → 仍失败则进入死亡流程
5. **死亡流程**：触发 `.夺舍重生` → 系统推送 3 个可夺舍目标 → 玩家 `.重生 <编号>` 选定 → 属性继承 + 境界跌落 + 冷却

辅助机制：
- **问道**（`.问道`）：元婴后期可向天道叩问，积累 `ask_dao_insight`，飞升成功率加成
- **法相天地**（`.法相天地`）：神通等级 0-9，每级提供 5% 全属性加成，飞升时每级额外 +2% 成功率
- **探寻裂缝**（`.探寻裂缝`）：每日限次探寻空间裂缝，可获得法则碎片 / 神识提升，但有神识反噬风险

#### 3.3.1 飞升前置条件检查（`AscensionService.checkPrerequisites`）

```javascript
 * 飞升前置条件检查

 * @returns {Object} { canAscend, reasons, bonuses }

async checkPrerequisites(playerId) {
    const player = await Player.findByPk(playerId);
    const ascension = await PlayerAscension.findOne({ where: { player_id: playerId } });
    const reasons = [];
    let bonuses = 0;

    // 1. 境界检查：必须化神巅峰（rank=20）
    if (player.realm_rank < 20) {
        reasons.push('境界未达化神巅峰');
    }

    // 2. 大衍诀层数：必须达到五层·衍神
    if (ascension.dayan_level < 5) {
        reasons.push(`大衍诀未达五层·衍神（当前 ${ascension.dayan_level} 层）`);
    }

    // 3. 逆灵通道坐标：必须已获得
    if (!ascension.reverse_channel_coord) {
        reasons.push('未获得逆灵通道坐标，需先 .搜寻节点 + .定星');
    }

    // 4. 空间法则碎片：至少 5 块
    if (ascension.law_fragments_count < 5) {
        reasons.push(`空间法则碎片不足（${ascension.law_fragments_count}/5）`);
    }

    // 5. 残魂值：必须 ≥ 80（防止残魂低时强行飞升）
    if (player.remnant_soul < 80) {
        reasons.push(`残魂值过低（${player.remnant_soul}/80），需先恢复`);
    }

    // 6. 状态检查：不能在闭关 / 战斗 / 副本 / 出窍中
    const stateCheck = await this.stateRegistry.checkAvailable(playerId, ['seclusion', 'combat', 'dungeon', 'soul_out']);
    if (!stateCheck.available) {
        reasons.push(`当前状态不可飞升：${stateCheck.reason}`);
    }

    // 计算成功率加成
    bonuses += player.ask_dao_insight * 0.001;       // 问道感悟：每点 +0.1%
    bonuses += player.dharma_form_level * 0.02;       // 法相天地：每级 +2%
    bonuses += ascension.dayan_level === 5 ? 0.10 : 0; // 大衍诀五层：+10%

    return {
        canAscend: reasons.length === 0,
        reasons,
        bonuses: Math.min(bonuses, 0.50) // 上限 +50%
    };
}
```

#### 3.3.2 飞升成功率公式

```
基础成功率 = 30%
境界rank补正 = (player.realm_rank - 20) * 0.02  // 已飞升玩家再飞升更高界域时
问道感悟补正 = ask_dao_insight * 0.001           // 上限 +10%（100点感悟）
法相天地补正 = dharma_form_level * 0.02          // 上限 +18%（9级）
大衍诀补正   = dayan_level === 5 ? 0.10 : 0       // +10%
残魂补正     = (remnant_soul - 80) * 0.005       // 80以上每点 +0.5%
法则碎片补正 = law_fragments_count * 0.01        // 每块 +1%

最终成功率 = min(0.95, 0.30 + 上述补正之和)
```

### 5.3 道侣关系建立与解除

#### 5.3.1 寻找道侣流程

```
玩家A 发起 .寻找道侣 @玩家B
       │
       ▼
系统校验：
  ├─ A、B 均无现有道侣
  ├─ A、B 境界均 ≥ 筑基期（rank≥4）
  ├─ A、B 不在闭关/战斗/副本中
  └─ A、B 互为好友（或同宗门，或同地图）
       │
       ▼
推送邀请给 B：「A 邀请你结为道侣」
       │
       ▼
B 选择 .我愿意
       │
       ▼
缔结道侣关系：
  ├─ 写入 dao_companion（state=active）
  ├─ A、B 玩家表 dao_companion_id 互填
  ├─ 双方获得称号「结发道侣」
  └─ 推送全服公告（可选）
```

#### 5.3.2 道侣解除机制

- **协议解除**：双方发送 `.斩断旧缘`，3 天冷静期后正式解除
- **毁誓解除**：单方发送 `.毁誓`，立即解除，但触发心劫惩罚（心契等级归零、修为损失 5%、24 小时虚弱）
- **死亡解除**：一方死亡夺舍后，道侣关系自动解除（夺舍后已是"另一个人"）

### 5.4 双修加成机制

#### 5.4.1 双修类型对比

| 类型 | 命令 | 修为加成 | 副作用 | 日上限 |
|------|------|---------|--------|--------|
| 闭关双修 | `.闭关双修` | 主元神 +5%，副元神 +3% | 无 | 3 次 |
| 温养 | `.双修 温养` | 双方各 +3%，且恢复气血 | 无 | 2 次 |
| 采补 | `.双修 采补` | 主方 +10%，副方 -3% | 副方残魂 -2，忠诚度要求高 | 1 次 |

#### 5.4.2 双修加成公式

```
单次双修修为 = 主元神当前修为 * 加成比例 * 心契等级系数 * 亲密度系数
心契等级系数 = 1 + heart_contract_level * 0.05   // 5级心契 +25%
亲密度系数   = 0.5 + (intimacy / 100) * 0.5       // 亲密度0时0.5倍，100时1.0倍
```

#### 5.4.3 双修前置条件

- 双方必须在线
- 双方必须在同一地图（或同一洞府）
- 双方均不在闭关 / 战斗 / 副本中
- 双方亲密度 ≥ 30（采补要求 ≥ 80）
- 双方残魂 ≥ 50

### 5.5 侍妾获取、培养、远航

#### 5.5.1 侍妾获取

- **红尘寻缘**（`.红尘寻缘`）：
  - 每日 1 次免费，额外 3 次消耗灵石（每次 5000 灵石，递增）
  - 概率获得侍妾（基础 5%，灵石加成最高 +15%）
  - 稀有侍妾概率 0.5%（如南宫婉、紫灵等）
- **副本掉落**：掩月抢亲通关有概率掉落侍妾（南宫婉）
- **宗门赐婚**：宗主可用 `.宗门赐婚` 给弟子赐婚侍妾
- **觉醒**：特定侍妾（如南宫婉）可用 `.唤醒婉影` 觉醒为高阶形态

#### 5.5.2 侍妾培养

#### 5.5.3 侍妾远航机制

**远航模式对比**：

| 模式 | 时长 | 风险 | 收益 | 奖励池 |
|------|------|------|------|--------|
| 稳妥 | 4 小时 | 极低 | 基础灵石 + 普通材料 | 灵石、低级丹药、灵草 |
| 均衡 | 8 小时 | 中 | 中等灵石 + 中级材料 | 灵石、中级丹药、稀有灵草 |
| 冒险 | 12 小时 | 高 | 高灵石 + 稀有材料 | 灵石、稀有法宝碎片、高级丹方 |
| 月殿寻痕 | 24 小时 | 极高 | 极高灵石 + 顶级材料 | 顶级法宝碎片、法则碎片、月华玉露 |

**远航成功率公式**：
```
成功率 = 0.5 + (charm / 100) * 0.2 + (intimacy / 100) * 0.15 + (loyalty / 100) * 0.15 - risk_modifier * 0.1
// 上限 95%，下限 5%
```

**远航奖励规则**：
- 单次最多 3 件稀有材料（防止刷爆）
- 失败时：保留 30% 灵石基础奖励，稀有材料全部消失
- 侍妾魅力 ≥ 80 时解锁月殿寻痕模式
- 远航期间侍妾不可问安 / 反哺 / 护法

**远航归来奖励领取**（`.远航归来`）：
- 必须在归来后 24 小时内领取
- 超时未领取则奖励消失，侍妾忠诚度 -5

### 5.6 心契 / 心印 / 心劫机制

#### 5.6.1 心契等级提升

| 3 | 共修 30 次 | 双修加成 +15%，心印上限 +1 |

| 5 | 立誓共修 50 次 | 双修加成 +25%，护道 30%，可种心印 2 个 |

#### 5.6.2 心劫触发条件

- 一方毁誓（`.毁誓`）
- 一方长期未问安（> 7 天）
- 一方死亡夺舍后
- 一方与第三方发生暧昧事件（侍妾亲密度 > 道侣亲密度）

#### 5.6.3 心劫事件流程

```

  │
  ▼

  ├─ 狠：成功率 50%，奖励亲密度 +20，残魂 -15
  └─ 骗：成功率 30%，奖励亲密度 +30，失败则心印破裂
  │
  ▼
结算：
  ├─ 成功 → 心契经验 +100，双方残魂恢复 +10
  └─ 失败 → 心契等级 -1，双方残魂 -20，24 小时虚弱
```

### 7.2 状态检查服务

在 `PlayerStateService` 中新增方法：

```javascript
/**
 * 检查玩家是否可进入指定状态
 * @param {number} playerId - 玩家ID
 * @param {string} newState - 欲进入的状态
 * @returns {Object} { available, reason, conflictingState }
 */
async canEnterState(playerId, newState) {
    const currentState = await this.getCurrentState(playerId);
    const config = this.stateRegistry.get(newState);
    if (config.exclusive.includes(currentState)) {
        return { available: false, reason: `当前${currentState}状态冲突`, conflictingState: currentState };
    }
    return { available: true };
}
```

## 十三、风险与边界处理

### 13.1 数据一致性风险

| 风险 | 应对措施 |
|------|---------|
| 飞升成功后玩家属性未同步 | 使用数据库事务，确保 `players` 与 `player_ascension` 同步更新 |
| 夺舍后境界跌落数值错误 | 在 `ReincarnationService` 中加双重校验：境界 rank 不能低于 1 |
| 双修时双方修为同时增加 | 使用悲观锁防止并发双修导致修为重复发放 |
| 多人副本成员状态不一致 | 副本实例操作走单一服务入口，所有成员状态变更走 `MultiDungeonInstanceService` |
| 香火产出溢出 | 定时任务每 5 分钟结算一次，单次产出有硬上限 |

### 13.2 经济平衡风险

| 风险 | 应对措施 |
|------|---------|
| 第二元神属性翻倍 | inherit_ratio 上限 0.6，且副元神境界比主元神低 1 大境界 |
| 小世界香火通胀 | 香火→神识兑换比 100:1，且每日淬炼上限 3 次 |
| 侍妾远航刷材料 | 单次最多 3 件稀有材料，远航归来 24 小时内必须领取 |
| 多人副本刷首通 | 首通仅全服首通奖励一次，副本冷却 7~23 小时 |
| 法则点刷爆 | 法则点日获取上限 50，强制长线积累 |

### 13.3 安全风险

| 风险 | 应对措施 |
|------|---------|
| SQL 注入 | 全部使用 Sequelize 参数化查询，禁止原生 SQL 拼接 |
| XSS 攻击 | 玩家自定义字段（小世界名、元神名、侍妾名）后端做 HTML 转义 |
| 越权操作 | GM 接口必须校验 `role=admin`，且记录 `admin_log` |
| 道侣关系伪造 | 道侣邀请必须双向确认，单方面不能强制缔结 |
| 副本实例劫持 | 副本实例校验队长身份，非队长无法推进抉择 |

### 13.4 边界场景处理

| 场景 | 处理逻辑 |
|------|---------|
| 玩家飞升时网络断开 | 飞升状态 10 分钟超时，超时自动判定失败 |
| 玩家夺舍时关闭浏览器 | 夺舍状态 30 分钟超时，超时强制随机选定目标 |
| 玩家在副本中掉线 | 成员 `is_present` 标记为 0，副本继续；玩家重连后可恢复 |
| 道侣双方同时发起双修 | 先到先得，后者收到"对方已在双修中"提示 |
| 侍妾远航期间被遣散 | 远航自动结算为 `interrupted`，奖励按 50% 发放 |
| 多人副本未满员开打 | 端午强制 10 人，掩月至少 3 人，否则拒绝开打 |
| 玩家在飞升时被 PVP 攻击 | 飞升状态独占 combat，无法被攻击 |
| 道侣一方死亡夺舍 | 道侣关系自动解除，另一方收到通知 |

### 16.1 设计文档结构总结

1. **飞升 + 夺舍重生系统**：13 张数据表中的 4 张（`player_ascension`、`player_ascension_node`、`reincarnation_target`、`player_reincarnation`）+ 完整状态机 + 数值公式
