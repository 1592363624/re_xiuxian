# 批次2：多人在线玩法深度设计方案

> 范围：世界BOSS系统 + 宗门战/领地争夺系统 + 已有 PVP/宗门数值平衡优化
> 设计目标：耐玩性、可玩性、多人协作与对抗深度
> 编写时间：2026-07-19
> 关联文档：`docs/功能对比清单.md` 第16节"世界事件与大型公共战斗"

---

## 一、设计哲学

### 1.1 核心调性：慢养成、强策略、多人协作

- **慢养成**：BOSS 与宗门战均按赛季/周期运营，奖励以"养成材料"为主而非"成品装备"
- **强策略**：BOSS 战需要"职业分工"（输出/治疗/承伤），宗门战需要"阵法/驻防/资源点"战术
- **多人协作**：单人无法击杀世界BOSS，必须组队；宗门战必须以宗门为单位参与

### 1.2 数值膨胀控制原则

| 维度 | 上限设计 |
|------|---------|
| 单场世界BOSS伤害占比 | 单人不超过 15%（防止大佬独占） |
| 单赛季宗门积分上限 | 100,000（按 7 天赛季 + 5 场战役计算） |
| 个人伤害阈值奖励 | 三档（1%/5%/15%），未达档位无奖励 |
| 宗门战参与人数上限 | 攻方 20 人，守方 30 人（守方有地形优势） |
| 资源点占领上限 | 单宗门最多占领 3 个（防垄断） |

### 1.3 多人交互设计要点

| 设计点 | 实现 |
|--------|------|
| **协作激励** | 伤害贡献奖励 + 参与奖 + 首杀成就 + 宗门积分 |
| **战术分工** | BOSS 阶段技能需要不同应对（破甲/集火/分散/承伤） |
| **赛季荣誉** | 个人伤害榜 + 宗门积分榜 + 赛季结算奖励 |
| **资源稀缺** | 顶级材料仅从世界BOSS/宗门战产出，形成玩家依赖 |
| **持续运营** | 周期性 BOSS 刷新 + 宗门战赛季，避免一次性内容 |

## 二、世界BOSS系统设计

### 2.1 玩法概述

参考修仙指南文档第16节"世界事件与大型公共战斗"，世界BOSS 是跨宗门的全服协作 PVE 玩法：

- **青元子**：元婴期BOSS，每周二/五/日 20:00 刷新
- **妖兽入侵**：化神期BOSS，每周一/四 20:00 刷新
- **慕兰烽烟**：炼虚期BOSS，每周三/六 20:00 刷新（赛季制）

每个BOSS有独立的：
- 阶段切换（HP 阈值触发技能变化）
- 群体技能（AOE/控制/召唤小怪）
- 专属掉落（材料/装备/法宝碎片）
- 伤害排行榜（个人 + 宗门）
- 首杀奖励（全服首杀 + 当日首杀）

### 2.3 BOSS战斗机制

#### 2.3.1 阶段切换设计

每个BOSS有3个阶段，根据当前HP百分比切换：

| 阶段 | HP 范围 | 技能变化 | 设计意图 |
|------|---------|---------|---------|
| 一阶段 | 100% ~ 60% | 普攻 + 单体技能 | 软开场，让玩家熟悉机制 |
| 二阶段 | 60% ~ 30% | 群体AOE + 召唤小怪 | 考验多人协作 |
| 三阶段 | 30% ~ 0% | 狂暴 + 全屏大招 | 考验承伤与治疗配合 |

#### 2.3.2 玩家攻击模式

- **单人挑战**：玩家可单人进入，但伤害大幅衰减（40%）
- **组队攻击**：3人以上组队，伤害正常计算，享受组队加成（+10%）
- **攻击CD**：每次攻击后冷却 5 秒，可被"鼓舞"道具降低
- **死亡与复活**：被BOSS击杀后扣 5% 修为，可选择：
  - 原地复活（消耗 1000 灵石，CD 60 秒）
  - 退出战斗（无奖励，5 分钟内不能再进入）

#### 2.3.3 战斗公式

```
单次伤害 = max(1, 玩家 ATK * 技能倍率 - BOSS DEF * 减伤系数) * 暴击系数 * 随机浮动
BOSS 反击伤害 = max(1, BOSS ATK * 阶段倍率 - 玩家 DEF * 减伤系数) * 技能系数
```

具体倍率配置在 `world_boss_data.json`：
- 阶段倍率：1.0 / 1.3 / 1.8
- 暴击概率：5%
- 暴击倍率：1.5
- 随机浮动：±15%

### 2.4 奖励机制

#### 2.4.1 伤害档位奖励（个人）

| 档位 | 伤害占比 | 奖励 |
|------|---------|------|
| 头部 | ≥ 15% | 顶级材料 ×3 + 灵石 ×5000 + 荣誉 ×200 |
| 中坚 | ≥ 5% | 高级材料 ×2 + 灵石 ×2000 + 荣誉 ×100 |
| 参与 | ≥ 1% | 中级材料 ×1 + 灵石 ×500 + 荣誉 ×30 |
| 边缘 | < 1% | 灵石 ×100 + 荣誉 ×10（参与奖） |

#### 2.4.2 宗门积分（多人激励）

宗门成员伤害占比之和 × 100 = 宗门积分，赛季末按积分排名发放：
- 第1名：宗门专属法宝 ×1 + 宗门资金 ×50000
- 第2-3名：宗门资金 ×30000
- 第4-10名：宗门资金 ×10000

#### 2.4.3 首杀奖励

- **全服首杀**：永久成就 + 顶级法宝 + 灵石 ×50000
- **当日首杀**：稀有材料 ×5 + 灵石 ×10000

#### 2.4.4 最后一击奖励

- 击杀者获得 "终结者" 称号 + 灵石 ×5000

### 2.6 定时调度设计

`WorldBossSchedulerService` 负责以下任务：

| 任务 | 频率 | 功能 |
|------|------|------|
| 刷新检查 | 每 1 分钟 | 检查是否到刷新时间，自动生成 BOSS |
| 过期清理 | 每 1 分钟 | 超过 expire_time 未击杀则自动消失 |
| 阶段切换 | 每 5 秒 | 检查 HP 是否进入下一阶段，触发技能 |
| 赛季检查 | 每日 00:05 | 检查赛季是否结束，触发结算 |

### 2.7 前端面板设计

`WorldBossPanel.vue`：
- 顶部：当前可挑战 BOSS 列表卡片（含倒计时）
- 中部：选中 BOSS 详情（HP 进度条、阶段指示器、技能预警）
- 右侧：实时伤害排行榜（个人 + 宗门）
- 底部：操作按钮（攻击/复活/撤退）
- WebSocket 推送：HP 实时更新、阶段切换提示、伤害浮动文字

## 三、宗门战/领地争夺系统设计

### 3.1 玩法概述

参考修仙指南文档第5节宗门玩法与第16节世界事件，宗门战是宗门间的大型 PVP 玩法：

- **赛季制**：每赛季 4 周，每周三、周六 20:00 开战
- **资源点争夺**：地图上有 9 个资源点（灵脉/矿脉/秘境入口），占领后持续产出
- **攻城战**：宣战 → 准备期 → 交战期 → 结算期
- **宗门积分**：根据占领资源点 + 战役胜负累积
- **赛季奖励**：按宗门积分排名发放宗门资金 + 专属法宝 + 荣誉

### 3.3 战役流程设计

#### 3.3.1 战役状态机

```
preparing（准备期，24小时）
  ↓ 宗主宣战（消耗 5000 灵石）
announced（宣战期，2小时，允许双方调兵遣将）
  ↓ active_start_time 到达
active（交战期，30分钟）
  ↓ active_end_time 到达 / 一方认输 / 全员阵亡
settled（结算期，立即结算并发放奖励）
```

#### 3.3.2 战斗机制

- **阵营制**：进入交战期后，参战玩家被分到攻方/守方阵营
- **复活机制**：阵亡后 60 秒可在己方出生点复活，无修为损失
- **战斗模式**：参战玩家可自由攻击对方阵营玩家，复用 `PvpService` 战斗公式
- **资源点占领**：
  - 攻方需在资源点停留 30 秒不被打断
  - 守方可设置驻防阵法（消耗宗门资金）
  - 占领成功后立即归攻方所有
- **战役胜负判定**：
  - 攻方积分 = 占领资源点数 × 100 + 击杀数 × 5
  - 守方积分 = 防守成功资源点数 × 100 + 击杀数 × 5
  - 积分高者胜，平局则守方胜（守方优势）

#### 3.3.3 资源点产出

每日 00:00 自动结算，按占领时长比例发放：
- 灵脉：每日 500 灵石/小时（宗门资金）
- 矿脉：每日 5 高级材料/小时
- 秘境入口：每日 100 贡献度/小时（全员均分）
- 战略点：无产出，但 +10% 战役胜率加成

## 六、关键技术决策

### 6.1 为什么不用 Redis 做伤害排行榜？

- 项目当前未引入 Redis，增加依赖会提高部署复杂度
- 单赛季参与人数预估 < 1000，MySQL 索引足够快
- 排行榜查询走 `ORDER BY total_damage DESC LIMIT 100` 索引，< 50ms

### 6.2 为什么 BOSS 战斗不实时同步全玩家HP？

- 同时参战玩家预估 < 100，BOSS HP 用 WebSocket 广播每 1 秒一次即可
- 玩家自身的伤害/受击即时反馈，BOSS HP 走"准实时"广播
- 减少服务端计算压力

### 6.3 为什么宗门战复用 PvpService 战斗公式？

- 保持战斗系统一致性，避免出现两套伤害计算
- 复用现有 `AttributeService.calculateFullAttributes` 计算属性
- 复用 `PlayerStateMachine` 状态互斥机制

### 6.4 状态机集成

新增两个互斥状态：
- `WORLD_BOSS`：参与世界BOSS战，与 SECLUDED/COMBAT/MOVING/ADVENTURE/PVP 互斥
- `SECT_WAR`：参与宗门战，与 SECLUDED/COMBAT/MOVING/ADVENTURE/PVP/WORLD_BOSS 互斥

### 6.5 WebSocket 推送通道设计

| 事件 | 通道 | 范围 |
|------|------|------|
| BOSS HP 更新 | `world_boss:hp_update` | 全服广播 |
| BOSS 阶段切换 | `world_boss:phase_change` | 全服广播 |
| BOSS 被击杀 | `notification:global` | 全服广播 |
| 个人伤害浮动 | `player:updated` (updateType=boss_damage) | 单玩家 |
| 战役状态变更 | `sect_war:state_change` | 双方宗门成员 |
| 战役分数更新 | `sect_war:score_update` | 双方宗门成员 |
| 资源点占领 | `sect_war:territory_change` | 全服广播 |

## 七、风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| 高并发下 BOSS HP 计算错误 | 用事务+行级锁，BOSS 行 `LOCK.UPDATE` |
| 玩家刷伤害（小号反复送死） | 死亡 5 次后强制退出战斗，30 分钟不能再进 |
| 宗门战参与人数不足 | 设置"最低参战人数"（5人），不足则战役取消 |
| 数据库迁移失败 | 每个迁移脚本幂等，使用 `INFORMATION_SCHEMA.COLUMNS` 检查 |
| 赛季结算时玩家正在战斗 | 结算前 5 分钟全服公告，禁止新战役发起 |
